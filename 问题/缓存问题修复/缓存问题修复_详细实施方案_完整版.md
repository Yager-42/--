## F1. 删除 `feed:card:stat:*` 二层缓存口径

**依赖**: S2 (决策确认) ✅

### 验收标准

- Feed/详情读取 `likeCount` 时不再创建或写入 `feed:card:stat:{postId}`

#### 步骤 1.2: 在 `ReactionCachePort` 实现批量接口

**文件**: `nexus-infrastructure/src/main/java/cn/nexus/infrastructure/adapter/social/port/ReactionCachePort.java`

**新增方法**:
```java
@Override
public Map<String, Long> batchGetCount(List<ReactionTargetVO> targets) {
    if (targets == null || targets.isEmpty()) {
        return Map.of();
    }

    // 1. 批量查 Redis
    List<String> keys = new ArrayList<>(targets.size());
    for (ReactionTargetVO target : targets) {
        keys.add(cntKey(target));
    }

    List<String> values;
    try {
        values = stringRedisTemplate.opsForValue().multiGet(keys);
    } catch (Exception e) {
        log.warn("batchGetCount redis multiGet failed", e);
        values = null;
    }

    Map<String, Long> result = new HashMap<>(targets.size());
    List<ReactionTargetVO> missTargets = new ArrayList<>();

    for (int i = 0; i < targets.size(); i++) {
        ReactionTargetVO target = targets.get(i);
        String raw = (values != null && i < values.size()) ? values.get(i) : null;
        Long v = parseLong(raw);
        if (v != null && v >= 0) {
            result.put(target.hashTag(), v);
        } else {
            missTargets.add(target);
        }
    }

    if (missTargets.isEmpty()) {
        return result;
    }

    // 2. 批量回表（miss 的才查 DB）
    for (ReactionTargetVO target : missTargets) {
        String targetType = target.getTargetType() == null ? null : target.getTargetType().getCode();
        Long targetId = target.getTargetId();
        String reactionType = target.getReactionType() == null ? null : target.getReactionType().getCode();

        if (targetType == null || targetId == null || reactionType == null) {
            result.put(target.hashTag(), 0L);
            continue;
        }

        Long dbCount = interactionReactionCountDao.selectCount(targetType, targetId, reactionType);
        long rebuilt = dbCount == null ? 0L : Math.max(0L, dbCount);

        // 回填 Redis
        try {
            stringRedisTemplate.opsForValue().set(cntKey(target), String.valueOf(rebuilt));
        } catch (Exception ignored) {
        }

        result.put(target.hashTag(), rebuilt);
    }

    return result;
}
```

**说明**：
- Redis 侧：1 次 `multiGet` 批量查询，消除 N 次 GET 的网络往返
- DB 侧：仅对 miss 的逐条 `selectCount` 回表（现有 DAO 只有单条接口，不新增批量 SQL）
- 冷启动后 Redis 命中率趋近 100%，DB 回表频率趋近 0

## F3. 统一 `ContentRepository.findPost` 协议

**依赖**: S2 (决策确认) ✅

### 验收标准

- `findPost` 走 Redis L2 + single-flight，不再旁路直接打 DB
- 命中 `"NULL"` 不回源
- 并发 miss 只允许一次回源
- `SingleFlight` key 必须带流程前缀，避免 `findPost` 与 `listPostsByIds` 复用同一个 `SingleFlight` 实例时发生 key 撞车
- 保持 `selectById` 的原 SQL 语义（查所有状态，无过滤）
- 只有 `findPost` 有资格写 NULL（因为它走 `selectById` 真相源）
- `listPostsByIds` 读到缓存后必须过滤 `status=2`，不能直接全收
- `listPostsByIds` 对 miss 的 id **禁止写 NULL**（因为 miss 可能只是"不是已发布状态"）
- 上线切换时必须处理历史假 `NULL`，不能让旧版 `listPostsByIds()` 写下的脏哨兵继续影响新版 `findPost()`

---

### 核心协议（必须严格遵守）

#### 协议 1：`interact:content:post:*` 的真义

**定义**：存储 `selectById` 查出来的"帖子真相对象"（所有状态，无过滤）

**写入资格**：
- ✅ `findPost()` 可以写（走 `selectById`）
- ❌ `listPostsByIds()` 不能写 NULL（走 `selectByIds` 只查 status=2）

**读取后处理**：
- `findPost()` 读到缓存后直接返回（无需过滤）
- `listPostsByIds()` 读到缓存后必须过滤 `status=2`

#### 协议 2：NULL 哨兵的写入资格

**只有以下情况才能写 `"NULL"`**：
- 走 `selectById` 真相源
- 确认 DB 真没有记录（不是"查不到已发布"，而是"真不存在"）

**禁止写 NULL 的情况**：
- `listPostsByIds()` 的 miss（可能只是 status != 2）
- 任何只查"已发布"的场景的 miss

#### 协议 3：`SingleFlight` key 必须命名空间隔离

**原因**：
- `ContentRepository` 内 `findPost()` 和 `listPostsByIds()` 复用同一个 `SingleFlight` 实例
- 如果两条链路都直接用 `String.valueOf(postId)` / `"1,2,3"` 这类裸 key，可能互相等待到错误类型的返回值

**强制规则**：
- `findPost()` 必须使用 `findPost:{postId}`
- `listPostsByIds()` 必须使用 `listPosts:{sortedIdsCsv}`
- 禁止继续使用裸 `postId` / 裸逗号串作为 `SingleFlight` key

#### 协议 4：历史假 `NULL` 的上线切换规则

**背景**：
- 旧版 `listPostsByIds()` 会把“未发布但真实存在”的帖子 miss 写成 `interact:content:post:* = "NULL"`
- 新版 `findPost()` 命中这个历史脏值后，会被误导成“帖子真不存在”

**本方案默认策略（必须执行）**：
- 发布前，批量删除 Redis 中所有 `interact:content:post:*` 且 value 为 `"NULL"` 的历史 key

**降级策略（只有批量删除做不到时才允许）**：
- 明确接受一个最多约 `30~40s` 的过渡窗口
- 在发布单里写明影响：极少数帖子可能被短暂误判为不存在，待旧 `NULL` TTL 自然过期后恢复
- 若采用降级策略，必须在发布后重点抽查详情页和 `findPost()` 调用链

> 结论：默认走“发布前清理历史假 NULL”，不要把风险留给线上自然过期。

```java
@Override
@Transactional(readOnly = true)
public ContentPostEntity findPost(Long postId) {
    if (postId == null) {
        return null;
    }

    // L1 热点缓存（保留原逻辑）
    boolean hot = isHotKeySafe(hotkeyKey(postId));
    if (hot) {
        ContentPostEntity cached = postCache.getIfPresent(postId);
        if (cached != null) {
            return copyPost(cached);
        }
    }

    // L2 Redis 缓存（新增）
    String redisKey = postRedisKey(postId);
    String cached = null;
    try {
        cached = stringRedisTemplate.opsForValue().get(redisKey);
    } catch (Exception e) {
        log.warn("findPost redis get failed, key={}", redisKey, e);
    }

    // 命中 NULL 哨兵，直接返回 null
    if (POST_REDIS_NULL_VALUE.equals(cached)) {
        return null;
    }

    // 命中正向缓存，解析并返回
    if (cached != null) {
        if (cached.isBlank()) {
            deleteRedisQuietly(redisKey);
        } else {
            ContentPostEntity entity = parsePostCache(cached);
            if (entity != null && entity.getPostId() != null) {
                if (hot) {
                    postCache.put(postId, copyPost(entity));
                }
                return copyPost(entity);
            }
            deleteRedisQuietly(redisKey);
        }
    }

    // single-flight 回源（新增）
    // 注意：必须带命名空间前缀，避免和 listPostsByIds() 共用 SingleFlight 时 key 撞车
    ContentPostEntity post = singleFlight.execute(findPostInflightKey(postId), () -> {
        // leader 内二次查 Redis（跨节点协调）
        String secondCheck = null;
        try {
            secondCheck = stringRedisTemplate.opsForValue().get(redisKey);
        } catch (Exception ignored) {
        }

        if (POST_REDIS_NULL_VALUE.equals(secondCheck)) {
            return null;
        }

        if (secondCheck != null && !secondCheck.isBlank()) {
            ContentPostEntity entity = parsePostCache(secondCheck);
            if (entity != null && entity.getPostId() != null) {
                return entity;
            }
            deleteRedisQuietly(redisKey);
        }

        // 回源：保持原 selectById 语义（查所有状态）
        ContentPostEntity rebuilt = toPostEntity(contentPostDao.selectById(postId));
        fillPostTypes(rebuilt == null ? List.of() : List.of(rebuilt));
        fillPostContent(rebuilt == null ? List.of() : List.of(rebuilt));

        // 写回 Redis
        if (rebuilt == null) {
            // 只有 selectById 确认不存在，才写 NULL
            try {
                stringRedisTemplate.opsForValue().set(
                    redisKey, POST_REDIS_NULL_VALUE, nullTtlSeconds(), TimeUnit.SECONDS);
            } catch (Exception ignored) {
            }
        } else {
            try {
                String json = serializePostCache(rebuilt);
                stringRedisTemplate.opsForValue().set(
                    redisKey, json, positiveTtlSeconds(), TimeUnit.SECONDS);
            } catch (Exception e) {
                log.warn("findPost redis set failed, key={}", redisKey, e);
            }
        }

        return rebuilt;
    });

    // 写入 L1（如果是热点且非空）
    if (hot && post != null) {
        postCache.put(postId, copyPost(post));
    }

    // 热点 TTL 延寿（与 listPostsByIds 对齐）
    if (post != null) {
        tryExtendHotCacheTtl(postId, post);
    }

    return post == null ? null : copyPost(post);
}
```

**关键点**：
- 保持 `selectById` 的原 SQL（查所有状态）
- 不复用 `listPostsByIds`
- 只加缓存层，不改语义
- 只有 `findPost` 有资格写 NULL
- `SingleFlight` key 必须带 `findPost:` / `listPosts:` 前缀，不能直接复用裸 `postId`
- 上线前默认清理历史假 `NULL`，不要把旧脏值留给新版 `findPost`
